# Logbook — 2026-07-15

## Entry 1: Full MVP Implementation (Phases 1–9)

**Date**: 2026-07-15

### Summary

Completed the full Centinela MVP implementation across all planned phases, from domain entities through composition root.

### Changes

#### Phase 1 — Domain + Ports + Services
- Retroactively cleaned `entities.py` to minimize `| None` usage: used empty strings and `id: int = 0` for DB-assigned IDs instead of `Optional` patterns. Only truly lifecycle-dependent fields retain None-like semantics via empty strings.
- Added `EmbeddingVector` and `SharpnessResult` entities for image pipeline.
- Created 12 port ABCs: `ProjectRepository`, `UserRepository`, `QueueRepository`, `HistoryRepository`, `InspectionRepository`, `HumanReviewRepository`, `TelegramGateway`, `ImageEmbedder`, `ImageProcessor`, `ClusteringEngine`, `LLMInspector`, `ReportGenerator`, `Clock`, `IdGenerator`.
- Created 8 services: `StartProjectService`, `FinishProjectService`, `IngestPhotoService`, `IngestMessageService`, `RegisterFloorplanService`, `HandleHITLResponseService`, `ProcessQueueItemService`, `prompt.py` (system/user prompt builder).

#### Phase 2 — SQLite Persistence
- `ext/shared/db.py`: WAL mode, foreign keys, 6-table schema (projects, users, queue, chat_history, inspections, human_reviews).
- 6 SQLite repository adapters implementing all ports.

#### Phase 3 — Telegram
- `ext/providers/ptb_telegram_gateway.py`: downloads files, sends messages/documents/keyboards via `python-telegram-bot`.
- `ext/controllers/telegram_bot.py`: Command handlers (`/iniciar`, `/finalizar`, `/plano`), photo handler, text handler, HITL callback query handler.

#### Phase 4 — Image Pipeline
- `PillowImageProcessor`: JPEG compression with configurable quality/max edge + OpenCV Laplacian variance for sharpness.
- `PhashImageEmbedder`: perceptual hash vectors via `imagehash`.
- `GreedyClusteringEngine`: simple distance-based clustering, picks sharpest representative.

#### Phase 5 — LLM Inspector
- `VllmLLMInspector`: multimodal analysis via `langchain-openai` + base64-encoded images + `guided_json` for structured output. Includes floor plan image when available.

#### Phase 6 — Queue Worker
- `ext/controllers/queue_worker.py`: async polling loop over active projects, processes oldest pending item per project.

#### Phase 7 — HITL
- Suspicious category triggers inline keyboard buttons (confirm/reject). Blurry images send notification and mark item as insufficient evidence.

#### Phase 8 — Report Generator
- `VllmReportGenerator`: generates Markdown report via LLM with inspection data + chat history context.

#### Phase 9 — Composition Root
- `src/__main__.py`: full DI wiring — creates all repos, providers, services, controllers, starts both the Telegram bot (long polling) and the queue worker as a concurrent asyncio task. Graceful shutdown.

#### Testing
- 13 fake implementations under `tests/fakes/`.
- 4 unit tests (start project, duplicate rejection, photo ingestion, message ingestion) — all passing.
