# Logbook — 2026-05-08

## Entry 1: vLLM deployment and end-to-end bot launch

**Date**: 2026-05-08

### Summary

Deployed vLLM on the AMD MI300X GPU droplet and launched the Telegram bot end-to-end.

### Details

1. **vLLM v0.17.1 (stable) failed** with Gemma 4:
   ```
   model type 'gemma4' but Transformers does not recognize this architecture
   ```
   The stable ROCm image (`vllm/vllm-openai-rocm:v0.17.1`) ships an older Transformers that lacks `Gemma4ForConditionalGeneration`.

2. **Switched to vLLM nightly** (`vllm/vllm-openai-rocm:nightly`). Image is ~30 GB, pull took ~15 minutes.

3. **Nightly (v0.20.2rc1.dev129)** successfully resolved the architecture:
   ```
   Resolved architecture: Gemma4ForConditionalGeneration
   ```

4. **Model weights** (58.25 GiB, bfloat16) loaded in ~22 seconds from the HuggingFace cache at `/root/.cache/huggingface/`. Container flags:
   ```bash
   --device /dev/kfd --device /dev/dri --shm-size=16g
   --dtype bfloat16 --max-model-len 16384
   --gpu-memory-utilization 0.90 --enforce-eager
   --trust-remote-code
   ```

5. **Inference verified** — `curl /v1/chat/completions` returned a correct response ("Hello") with `finish_reason: stop`.

6. **Telegram bot** started locally (`PYTHONPATH=src:. python src/__main__.py`), connected to `@C3nt1nel_bot`, long polling active.

### Decisions

- Use `vllm/vllm-openai-rocm:nightly` until Gemma 4 support lands in a stable release.
- Alternatives identified if nightly breaks: **SGLang** (ROCm), **HF Inference Providers** (Together AI, DeepInfra), **llama.cpp GGUF** (192 quantised variants on HF).

### Artifacts

- `scripts/start_vllm.sh` updated with `VLLM_IMAGE=vllm/vllm-openai-rocm:nightly`.
- `containers/Containerfile.bot` added for future cloud deployment of the bot.
